Style guides are the new Photoshop

- Stephen Hay - Twitter: @stephenhay - Video: http://vimeo.com/52851510

I liked Stephen's chapter in Smashing book #3, so I was really looking forward to see this presentation. Stephen's presentation is about changing the process of building and designing systems.

His first advise is to manage clients expectations by designing in the browser instead of creating a static design in Photoshop. This way the customer learns to live with constraints.

The disadvantages of Photoshop

What are the disadvantages of Photoshop, according to Stephen Hay?

  • Responsive design implies multiple and flexible layouts.
  • This is too much manual work.
  • And it makes the image editor as a dependency.
  • Design changes are therefore time consuming.

The advantages of Web technology

  • It is very practical for responsive design mockups
  • Making mockups in web tech gives a realistic presentation. Which also helps to manage the clients expectations.
  • It is free and open: everyone has a browser & editor
  • No negative surprises for the client
  • Possible preperation for development
  • And they can be responsive
Stephen: “What is it with designers that they are scared for the command line?”
Audience: “It's not natural. No GUI!”
Stephen: “Photoshop is pretty difficult too, for something like the LAB colorspace there are books of > 600 pages!”

We need two things to replace Photoshop for mock ups. Number one, consider Photoshop not as a design tool for a complete mock up. Number two, use style guides.

The advantages of style guides

  • No measuring, with looking at Photoshop and doing math for width etc.
  • State and breakpoint changes can be included.
  • Useful for future designers, developers & others.
  • Design consistency and maintainability.

See for some examples of style guides written by Anna Debenham. And the BBC GEL.

What tools do you need for style guides?

Stephen Hay has a wish list for the style guide approach:

  • It should be free-form writable
  • Taking screenshots should be automated
  • Update code snippets automatically
  • Automatically update screenshots after change
  • Elements/modules should not have to be physically split into separate files.
  • Syntax highlight

There are tools available on the internet which can fullfill these wishes:

  • Dexy, which you can use for free-form documentation. It is the most flexible documentation tool ever.
  • Phantomjs and Casperjs for updating screenshots.
  • Jinja. For code snippets. It is also a full featured template engine for Python.
  • Pygments for syntax highlight. The "scary" part is that it involves the command line.

He also states that the command line is not a scary thing. Photoshop is the most difficult and extensive program. A few command lines shouldn't be so hard to learn.

Steps for using style guides

  1. Create web based mock up:
    Your goal is not to create production code when creating a web based mock up. The goal is to represent the design in the browser. Your CSS should be modular because it will end up in your style guide.
  2. set up style guide:
    Install dexy, phantom js, caspar js
  3. Write your guide and use jinja templates.
  4. All the CSS for your site should be modular, e.g. using a technique like SMACSS
  5. Mark the css you want to export
  6. Run Dexy
